webpack ：模块化打包工具

  1. cnpm install webpack -g
  2. cnpm init
  3. cnpm install webpack --save
  4. 直接写路径导出：webpack src/js/index.js assets/js/index.js

配置工具：

  1. 根目录建立一个文件webpack.config.js
  2.     			module.exports={    	 
    			   entry: __dirname + "/src/js/index.js",
    			   output: {
    			   	path: __dirname + "/assets/js",
    			   	filename: "index.js"
    			   }
    	        }
  3. webpack
    1. 另外一种方法：
    2. package.json文件里面： 
              		   	 "scripts": {
        		   	   "start": "webpack"
        		   	 }
        		   	 cnpm start

注意：

  • gulp优化了前端工作流
  • 使用webpack时，文件夹不能用webpack命名，会报错
  • 前端构建工具的骨架：
    1. 开发环境(src文件夹>css,js,images,lib(外部引用的库（jq,vue,react）))
    2. 测试环境(db)
    3. 生产环境（assets(Android环境下此文件夹不被编译)文件夹）
  • 路径中斜杠不能少
  • 当script名称是start时，直接cnpm start,其它都要加run,例如：cnpm run build
