适用场景：

  1. 高并发性
  2. 以服务方式海浪化数据
  3. webapp
  4. 消息列队

不适用场景：

  1. 高cpu使用率
  2. 简单web应用（开个虚拟主机就好）
  3. 大量回调式编程逻辑

node.js是单线程非阻塞式模型（不妨碍拿数据，拿完很多个数据后一个个处理）

  1. 线程的大哥是进程（开很多个nodejs）
  2. c++自己释放内存，node通过c++释放内存

为什么选择nodejs？

  1. 高性能web服务器
  2. 大前端改造了互联网革命
  3. 简单易学

npm命令:

  1. npm -v：查看版本号
  2. npm root：查看包的命令
  3. npm root -g：查看全局安装包的命令
  4. npm install：安装命令
  5. npm install -g：全局安装
  6. npm init：package.json
  7. npm ls：显示所有安装包
  8. npm remove：删除
  9. npm update：更新
  10. npm help：查看npm所有命令

node 编码规范：

  1. 加var声明
  2. 加减乘除等运算符前后加空格
  3. +-*/前后用空格
  4. 全用单引号
  5. 逗号放在后面
  6. 分号（一句话结束后一定要用分号）
  7. 变量命名（小驼峰写法） 不能这样写（student_name）,只能这样写studentName
  8. 方法命名 第一个单词尽量用动词，例如getStudentId returnUserInfo
  9. 类命名（大驼峰写法）UserDatabase
  10. 常量命名(大写 下划线)：FORM_NAME
  11. 包名 不要写：js node
  12. node里面尽量不要使用三个等号，建议两个等号
  13. 异步加载：function sss(err,data){}
